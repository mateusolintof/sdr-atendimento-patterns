Conectar a Evolution API ao Chatwoot

Ligue o WhatsApp de um cliente a um painel omnichannel. Ao final, toda conversa cai sozinha numa caixa de entrada do Chatwoot — sem automação, sem IA, sem código.

Evolution API 2.3.7 Chatwoot 4.13.0 ⏱ ~15 min 🧩 17 passos 👤 Dono: [preencher] 🔁 Revisão: [preencher]

▸ Quando usar

Entrou um cliente novo e você precisa que as mensagens de WhatsApp dele apareçam e fiquem registradas numa caixa de entrada do Chatwoot (contatos + conversas). Use este runbook uma vez por cliente. Não cobre automações nem IA — isso é um segundo momento.

Antes de começar Pré-requisitos

Tenha tudo isto em mãos. Faltando um, pare aqui e resolva antes — no meio do fluxo é pior.

  • Acesso superadmin ao Chatwoot — login em chat.almaconvert.com.br com permissão de Super Admin.
  • Acesso ao Evolution Manager — evo.almaconvert.com.br/manager + a API Key Global (com o responsável técnico).
  • Celular do cliente em mãos — com o WhatsApp que vai ser conectado, para ler o QR Code.
  • Nome definido — como a instância e a caixa de entrada vão se chamar (ex.: instituto-igor).

Bloco 1 · Chatwoot Preparar a conta e pegar as chaves

Você vai criar a conta do cliente e sair daqui com dois valores que a Evolution precisa: o Account ID e o Access Token.

1

Entre no Chatwoot como superadmin

Navegador → chat.almaconvert.com.br

Abra o endereço, digite seu e-mail e senha de superadmin e clique em Entrar.

Esperado: o painel do Chatwoot abre com a lista de conversas.

Por quê: só o superadmin consegue criar contas novas de cliente no passo seguinte.
2

Abra o Super Admin Console

Menu do seu perfil → Console do Super Admin · ou /super_admin

Clique no seu avatar (canto inferior esquerdo) e escolha Console do Super Admin.

Esperado: abre o painel de administração com o menu Accounts (Contas).

Por quê: é a área onde ficam todas as contas de cliente. Cada cliente = uma conta isolada.
3

Crie a conta do cliente

Super Admin → Accounts → New Account

Clique em New Account, digite o nome do cliente (ex.: Instituto Dr. Igor) e salve.

Esperado: a nova conta aparece na lista de contas.

Por quê: isola as conversas desse cliente das dos demais. Uma conta por cliente, sempre.
4

Confirme um usuário Administrator na conta

Dentro da conta → Configurações → Agentes

Entre na conta criada. Em Configurações → Agentes, garanta que existe (ou adicione) um usuário com a função Administrator.

Esperado: pelo menos um agente com a marca Administrator na lista.

Por quê: o token que você vai copiar no passo 6 é o desse usuário. A integração age em nome dele — e ela precisa de permissão de admin para criar a caixa de entrada.
5

Anote o Account ID pela URL

Barra de endereço → /accounts/N/…

Com a conta aberta, olhe a barra de endereço. O número depois de /accounts/ é o Account ID. Anote-o.

Esperado: você tem o número (ex.: 2). É o N que a Evolution vai pedir.

Por quê: é ele que diz à Evolution em qual conta as conversas devem cair. Cada cliente tem o seu.
6

Copie o Access Token do seu perfil

Avatar → Configurações do Perfil → final da página, Token de acesso

Abra Configurações do Perfil, role até o fim e clique em Copiar no bloco Token de acesso.

Esperado: o token fica na área de transferência. Guarde-o num lugar seguro.

Trate o token como uma senha. Não cole em conversa, e-mail ou print. Ele dá acesso à conta.
Por quê: é a credencial que autentica a Evolution na API do Chatwoot.

Bloco 2 · Evolution Conectar o WhatsApp e ligar a ponte

Agora no Evolution Manager: criar a instância, conectar o número e preencher a integração com os dois valores que você anotou.

7

Entre no Evolution Manager

Navegador → evo.almaconvert.com.br/manager

Abra o Manager. O campo Server URL já vem preenchido. Cole a API Key Global e clique em Login.

Esperado: abre a tela Instances (lista de instâncias).

Você mesmo digita a API Key Global — é uma chave sensível. Peça-a ao responsável técnico; não a compartilhe.
8

Crie a instância do cliente

Instances → botão Instance + → modal New Instance

Clique em Instance +. Em Name, use o nome do cliente. Deixe Channel em Baileys e clique em Save.

Esperado: a instância aparece na lista, com status Disconnected (ainda sem WhatsApp).

Por quê: a instância é o "ramal" da Evolution que representa o WhatsApp desse cliente. Baileys é o modo que conecta um número via QR Code.
9

Abra o QR Code da instância

Instances → clique no ⚙️ ao lado do nome → QR Code

Na lista, clique no ⚙️ ao lado da instância para abrir os detalhes. Um QR Code aparece na tela.

Esperado: um QR Code grande, pronto para ser escaneado.

Não compartilhe esse QR. Quem o escanear conecta o WhatsApp do cliente. Use-o na hora e feche.
10

Escaneie o QR pelo WhatsApp do cliente

Celular → WhatsApp → Aparelhos conectados → Conectar aparelho

No celular do cliente: WhatsApp → Aparelhos conectados → Conectar um aparelho → aponte para o QR.

Esperado: o status da instância muda para Connected (verde).

Por quê: é o que autoriza a Evolution a enviar e receber mensagens por aquele número. Sem isto, nada trafega.
11

Abra a integração Chatwoot da instância

Detalhes da instância → menu lateral → Integrations → Chatwoot

No menu lateral da instância, abra Integrations e clique em Chatwoot.

Esperado: abre a tela Chatwoot com o formulário de configuração.

Por quê: é aqui que a Evolution aprende a "conversar" com o Chatwoot da conta que você preparou.
12

Preencha os campos da integração

Tela Chatwoot → de cima para baixo

Ligue Enabled e preencha os campos com os valores da tabela abaixo. Os principais são Chatwoot URL, Account ID e Token (o que você copiou no passo 6).

Na Chatwoot URL, não deixe barra / no final. Isso quebra a integração.

Esperado: todos os campos preenchidos, sem salvar ainda (o Save é o passo 13).

13

Ligue o Auto Create e salve

Final da tela Chatwoot → Auto Create + Save

Role até o fim, ligue Auto Create e clique em Save.

Esperado: mensagem de sucesso. A caixa de entrada é criada sozinha no Chatwoot.

Por quê: o Auto Create faz a Evolution criar a caixa de entrada (tipo API) e configurar o retorno das mensagens. Por isso você não cria canal manualmente no Chatwoot.

Bloco 3 · Verificação Provar que funcionou

Quatro checagens rápidas. Se as quatro passarem, o omnichannel está no ar.

14

Confirme a caixa de entrada no Chatwoot

Chatwoot → Configurações → Caixas de entrada

Volte ao Chatwoot da conta e abra Configurações → Caixas de entrada.

Esperado: aparece a caixa com o nome que você definiu, do tipo API.

Por quê: é a prova de que o Auto Create funcionou.
15

Teste o recebimento

De um WhatsApp qualquer → para o número do cliente

De um celular de teste, mande uma mensagem para o número do cliente.

Esperado: em segundos, surge uma conversa nova na caixa de entrada.

16

Teste o envio

Chatwoot → responder na conversa

Responda a essa conversa pelo Chatwoot e envie.

Esperado: a resposta chega no WhatsApp do celular de teste.

Por quê: valida o caminho de volta (o retorno que o Auto Create configurou).
17

Confirme o contato único

Chatwoot → Contatos

Abra Contatos e procure o número de teste.

Esperado: um contato só — não dois (com e sem o 9º dígito).

Por quê: confirma que o Merge Brazil Contacts está fazendo efeito.

Checklist Verificação final

  • ✓
    Caixa de entrada criada — aparece em Configurações → Caixas de entrada, tipo API.
  • ✓
    Recebe mensagem — WhatsApp de teste vira conversa nova.
  • ✓
    Envia mensagem — resposta pelo Chatwoot chega no WhatsApp.
  • ✓
    Sem contato duplicado — um único registro por número.

Se travar Problemas comuns

▸ Recebe, mas não envia

Quase sempre é a Chatwoot URL com "/" no final, o Token sem permissão de admin, ou o Auto Create desligado. Volte ao passo 12/13, corrija e clique em Save de novo.

▸ Nada chega na caixa

Verifique se a instância está Connected (passo 10), se o Account ID é o número certo da URL (passo 5) e se o Enabled está ligado (passo 12).

▸ Contatos duplicados

O Merge Brazil Contacts está desligado. Ligue-o no passo 12 e salve. Os próximos contatos deixam de duplicar.

Desfazer Rollback

Precisa reverter? Desligar a integração interrompe o espelhamento na hora — sem apagar o que já foi salvo.

  1. Desligue o Enabled na tela Chatwoot da instância e clique em Save. Isso interrompe o espelhamento imediatamente.
  2. (Opcional) Exclua a caixa de entrada criada, em Configurações → Caixas de entrada.
  3. (Opcional) Desconecte ou exclua a instância no Evolution Manager.

ℹ Nota

As conversas e contatos já registrados permanecem no Chatwoot. O rollback só corta o fluxo novo — não apaga histórico.
